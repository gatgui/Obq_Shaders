Obq_Simbiont


Class

Texture [Generator]


Output

Color RGBA


Synopsis

Obq_Simbiont is a fractal generator node that read DarkTree files.

Note : Check the installation guide of the documentation to properly install the needed library.


Fig.1 : A simple DarkTree file with displacement workflow. Use Elevation as Alpha output and a picker to select it and plug into the displacement port.


Usage

Browse for a DarkTree file. Select the type of output for RGB and Alpha. Adjust the options and tweaks et voillà.


DarkTree


Darktree file [filename]

The darktree file you want to load.

Note : You can change the default browsing directory on line #660 of the Obq_Simbiont.spdl.


DarkTree component libs Path [componentLibsPath]

This is the path to the DarkTree Engine "dte_components" libraries. If this path is incorrect, you will see red Xs as the resulting texture.

Note : This path is define on line #30 of the Obq_Simbiont.spdl.


Preserve tweaks on load [preserveTweaksOnLoad]

This doesn't reload tweaks when filename_OnChange() is called. This can be useful for scripting purpose.


Output


RGB [evaluateRGB]

A choice of output for the RGB channels :

  • Black
  • Color [default]
  • Percent
  • Elevation [displacement or bump]
  • Diffuse level
  • %enumItem
  • Luminosity
  • Specular Level
  • Specular Func
  • Specular Color
  • Glossiness
  • MetalLevel
  • Anisotropy
  • Aniso Dir
  • Reflectivity
  • Environement
  • Transparency
  • Refraction
  • Clear Coat Level
  • Clear Coat Glossiness
  • Clear Coar Thickness
  • Clear Coat Smoothness
  • Bump [Normal map]
  • Alpha

Grayscale [outRRR]

Copies the red channel into blue and green for grayscale values.


Alpha [evaluateAlpha]

A choice of output for the alpha channels, the choices are the same as RGB parameter.


Bump


Sample Size [sampleSize]

A rough indication of the diameter of the point/pixel being sampled. This is used when evaluating bump.


Bump Scale [bumpScale]

A bump height scaling factor. This is used when evaluating bump.


Camera


Camera Near [cameraNear]

Some darktree files have effects based on a normalized value of how far is the camera.

Note : Use the near and far to give the range for the effect.


Camera Far [cameraFar]

Same as the above.


Others


Continuous Rep [continuousRep]

A 0.0 to 1.0 normalized value used for function evaluation. DTE_EVAL_DIFFUSE_FUNC and DTE_EVAL_SPECULAR_FUNC are where this value is often used when evaluating a texture. If you need additional information on using the specular and diffuse functions, please contact DarkSim support.


Region Rep [regionRep]

A random integer value used to indicate different regional areas of the texture. Patterned components within DarkTree, Bricks for example, set this value for linked child components. Control components use this value to generate random colors and values. A potential use for this value would be a custom designed texture that requires regional information to be passed into the texture from the renderer.


Animation


Time [frameNumber]

Can be seen as a current frame number for an animated texture. This is a floating point value and need not be an integral frame. Please note that this will have an effect only on DarkTrees that use the Time component.


Coordinates


Coordinates System [coordinatesSystem]

Coordinates system used for evaluation :

•Object Space:Does not change with transforms or deformation.
•World Space:Changes with transforms or deformation.
•UV Space:Active UV set.
•Camera Space:Camera projection.
•Raster Space:Pixel space.
•Custom:A custom vector input (customCoordinates parameter).

Custom Coordinates [customCoordinates]

A custom coordinates input. This is most often used to connect a node such as a "Projection Lookup".


Transform


Scale [coordinateScale]

This is a global scaling factor.

Note : If you want to scale down the fractal, you have to scale up this parameter.


Acceleration


Tweak at the global scope [globalTweaksOnly]

Tweaks are perform with the given value for all the object. This is good when you don't need to drive tweaks with textures. Otherwise, it should be off.


Tweak Actions

These buttons allows you the save, load and reset tweaks with a simple text file.


Tweaks #

Tweaks allow someone to change certain values of the DarkTree file on the fly.

Note : A given DarkTree file may contain any number of macro controls, or Tweaks. These tweaks are evaluated at runtime and can be modified with textures in Softimage. There is a maximum of 16 tweaks in this implementation. The tweaks are automatically loaded by a python script when the filename is change. Only the proper input choice will be displayed.

•Enable:Enable this tweak.
•Name:The name of the tweak.
•Color:A color input for the tweak. Used only for color driven tweaks.
•Float:A scalar input for the tweak. If the tweak is of boolean nature, x >0 => true. If the tweak requires an integer, the scalar will be rounded.
